# Initial IT Venture Idea Log

## Idea 1: Smart Campus Lost-and-Found System

### Problem Area
Students often lose items on campus and do not know where to report or search for them.

### Target Users
University students, security staff, and administrative staff.

### Current Alternative
Students usually post in chat groups or ask security manually.

### Proposed IT Solution
A web-based lost-and-found platform where users can report found items, search missing items, and contact the responsible office.

### Possible Technology
HTML/CSS/JavaScript, Google Sheets database, image upload feature.

### Why This Is Suitable
Can be built as a simple web application within one semester.

## Idea 2: Student Study Group Finder

### Problem Area
Students struggle to find classmates interested in studying together.

### Target Users
University students.

### Current Alternative
LINE groups and social media posts.

### Proposed IT Solution
A platform that matches students based on courses and study interests.

### Possible Technology
HTML/CSS/JS, Firebase.

### Why This Is Suitable
Simple MVP with user registration and matching features.

---

## Idea 3: Campus Event Notification System

### Problem Area
Students often miss important university events and deadlines.

### Target Users
University students and clubs.

### Current Alternative
Checking multiple social media pages.

### Proposed IT Solution
A centralized dashboard for campus events and reminders.

### Possible Technology
Web application with notification features.

### Why This Is Suitable
Easy to prototype and validate with students.
